# discord-api-practice

## 概要

Discord Webhookを使い、ターミナルで入力した内容を見やすいEmbed形式でDiscordチャンネルへ送信する通知ツールです。

## 主な機能

- ターミナルから通知タイトルと通知本文を入力できます。
- 「通常」「重要」「緊急」の3段階から重要度を選択できます。
- DiscordへEmbed形式で通知します。
- 重要度に応じて絵文字、表示名、Embedの色が変わります。
  - 🟢 通常
  - 🟠 重要
  - 🔴 緊急
- 日本時間（Asia/Tokyo）の送信日時を自動で表示します。
- 送信前にタイトル、本文、重要度を確認できます。
- 確認時に`y`を入力すると送信し、`n`を入力するとタイトルから再入力できます。
- タイトルまたは本文の入力時に`exit`か`quit`を入力すると、送信せず終了します（大文字・小文字は区別しません）。
- タイトルは最大256文字、本文は最大4096文字です。
- APIエラー、通信エラー、タイムアウトの内容を日本語で表示します。

## 使用技術

- Python
- requests
- python-dotenv
- Discord Webhook
- Python標準ライブラリのdatetime / zoneinfo

## ファイル構成

```text
discord-api-practice/

├── .env.example

```

## セットアップ方法

1. 仮想環境を作成します。

```bash
python3 -m venv .venv
```

2. 仮想環境を有効化します。

```bash
source .venv/bin/activate
```

3. 必要なライブラリをインストールします。

```bash
pip install -r requirements.txt
```

4. `.env.example`をコピーして、`.env`ファイルを作成します。

```bash
cp .env.example .env
```

作成した`.env`にDiscordのWebhook URLを設定してください。Webhook URLはコードやREADMEへ直接記載しないでください。

## 実行方法

以下のコマンドでプログラムを実行します。

```bash
python3 discord_notify.py
```

## 操作例

```text
通知タイトルを入力してください：サーバーメンテナンス
通知本文を入力してください：本日22時からメンテナンスを実施します。
重要度を選択してください。
1：通常
2：重要
3：緊急
選択：2

送信する通知
--------------------
タイトル：サーバーメンテナンス
本文：本日22時からメンテナンスを実施します。
重要度：🟠 重要
--------------------
この内容で送信しますか？（y/n）：y
ステータスコード：204
Discord通知の送信に成功しました
重要度：🟠 重要
送信日時：2026年7月10日 21:30:00
```

確認画面で`n`を入力すると、送信せずタイトル入力からやり直します。タイトルまたは本文に`exit`か`quit`を入力すると通知をキャンセルします。

## 入力制限

- タイトル：1～256文字（空欄や空白だけの入力は不可）
- 本文：1～4096文字（空欄や空白だけの入力は不可）
- 重要度：`1`、`2`、`3`のいずれか
- 送信確認：`y`または`n`

## エラーが表示された場合

Webhook URLが未設定の場合は、`.env`に`DISCORD_WEBHOOK_URL`が設定されているか確認してください。送信失敗時にはステータスコードとDiscordからのエラー内容が表示されます。通信エラーやタイムアウトの場合は、ネットワーク接続を確認してから再実行してください。

## セキュリティ上の注意

- `.env`には本物のDiscord Webhook URLを保存するため、GitHubへ絶対に公開しないでください。
- `.env`をGitへ追加またはコミットしないでください。
- このプロジェクトでは`.gitignore`に`.env`を追加し、Gitの管理対象から外しています。この設定を削除しないでください。
- `.env.example`には本物のWebhook URLを書かず、設定例だけを記載してください。
- Webhook URLをコード、README、ログ、画面出力へ記載しないでください。
